Proto reference

Every proto that appears anywhere in this stack. Tabs select by file. Each message is annotated.

resource.proto public

The user-facing identifier for any cache blob. Comes "in the door" from a Bazel client.

proto/resource.proto
message ResourceName {
  build.bazel.remote.execution.v2.Digest digest = 1;
  string instance_name = 2;
  build.bazel.remote.execution.v2.Compressor.Value compressor = 3;
  CacheType cache_type = 4;   // AC (action cache) or CAS (content addressable storage)
  build.bazel.remote.execution.v2.DigestFunction.Value digest_function = 5;
}

enum CacheType { UNKNOWN_CACHE_TYPE = 0; AC = 1; CAS = 2; }

Used everywhere in metacache.go — Get, Set, Contains, Reader, Writer all take a *rspb.ResourceName. The metacache translates it to a FileRecord (next tab) via makeFileRecord.

storage.proto internal

The persisted, internal form. FileRecord is the key, FileMetadata is the value, and StorageMetadata is the discriminated union telling the system where the bytes live.

Isolation — namespacing for a record
message Isolation {
  resource.CacheType cache_type = 1;
  string remote_instance_name = 2;
  string partition_id = 3;
  string group_id = 4;
}
FileRecord — the canonical key
message FileRecord {
  Isolation isolation = 1;
  build.bazel.remote.execution.v2.Digest digest = 2;
  build.bazel.remote.execution.v2.Compressor.Value compressor = 3;
  Encryption encryption = 4;
  build.bazel.remote.execution.v2.DigestFunction.Value digest_function = 5;
}
StorageMetadata — the discriminated union
message StorageMetadata {
  message FileMetadata    { string filename = 1; }
  message PebbleMetadata  { bytes key = 1; int64 chunks = 2; }
  message InlineMetadata  { bytes data = 1; int64 created_at_nsec = 2; }
  message GCSMetadata     { string blob_name = 1; int64 last_custom_time_usec = 2; }
  FileMetadata    file_metadata    = 1;
  PebbleMetadata  pebble_metadata  = 2;
  InlineMetadata  inline_metadata  = 3;
  GCSMetadata     gcs_metadata     = 5;
}

The metacache only uses InlineMetadata and GCSMetadata. The other two variants are used by other cache backends (PebbleCache, file-based caches) which do not currently route through the raft metadata service.

FileMetadata — the value at each key
message FileMetadata {
  FileRecord file_record = 1;
  StorageMetadata storage_metadata = 2;
  EncryptionMetadata encryption_metadata = 6;
  int64 stored_size_bytes = 3;       // what was actually written
  int64 last_access_usec = 4;        // atime; bumped by reads via UpdateAtimeRequest
  int64 last_modify_usec = 5;
}
PartitionMetadata — global per-partition usage counters
message PartitionMetadata {
  int64 size_bytes = 1;
  int64 cas_count = 2;
  int64 ac_count = 3;
  int64 total_count = 4;
  string partition_id = 5;
}

metadata.proto public RPC types

Tiny — wrappers for the four operations.

Get / Set / Delete / Find request & response types
message GetRequest    { repeated storage.FileRecord file_records = 1; }
message GetResponse   { repeated storage.FileMetadata file_metadatas = 1; }

message SetRequest {
  message SetOperation { storage.FileMetadata file_metadata = 1; }
  repeated SetOperation set_operations = 1;
}
message SetResponse {}

message DeleteRequest {
  message DeleteOperation {
    storage.FileRecord file_record = 1;
    int64 match_atime = 2;     // optimistic CAS: only delete if atime matches
  }
  repeated DeleteOperation delete_operations = 1;
}
message DeleteResponse {}

message FindRequest  { repeated storage.FileRecord file_records = 1; }
message FindResponse {
  message FindOperationResponse {
    bool present = 1;
    int64 last_access_usec = 2;
  }
  repeated FindOperationResponse find_responses = 1;
}
match_atime is a quiet superpower
The optional match_atime on each DeleteOperation is what lets eviction be safely concurrent with reads/writes. The eviction sampler observes a row's atime; if anyone touches the row before the delete arrives, the conditional fails and the row stays. Used by the usagetracker eviction pipeline and by metacache.handleMetadataMismatch for orphan cleanup.

metadata_service.proto public gRPC

The entire public service. Only four methods.

proto/metadata_service.proto
service MetadataService {
  rpc Get(metadata.GetRequest)       returns (metadata.GetResponse);
  rpc Set(metadata.SetRequest)       returns (metadata.SetResponse);
  rpc Delete(metadata.DeleteRequest) returns (metadata.DeleteResponse);
  rpc Find(metadata.FindRequest)     returns (metadata.FindResponse);
}

A metadata_server binary (enterprise/server/cmd/metadata_server/main.go) registers this server. Clients are the various metacache.Cache instances elsewhere in the fleet.

raft.proto internal

734 lines, four conceptual sub-sections. This is what flows over dragonboat as state-machine commands.

1 · Storage & wire

message KV { bytes key = 1; bytes value = 2; }

message RequestUnion {
  oneof value {
    DirectWriteRequest  direct_write = 1;
    DirectReadRequest   direct_read  = 2;
    DirectDeleteRequest direct_delete= 15;
    IncrementRequest    increment    = 3;
    ScanRequest         scan         = 4;
    CASRequest          cas          = 5;
    FindSplitPointRequest  find_split_point = 14;
    DeleteSessionsRequest  delete_sessions  = 16;
    FetchRangesRequest     fetch_ranges     = 17;
    GetRequest    get    = 7;
    SetRequest    set    = 8;
    DeleteRequest delete = 9;
    FindRequest   find   = 10;
    UpdateAtimeRequest update_atime = 11;
  }
}

message BatchCmdRequest {
  Header header = 1;
  repeated RequestUnion union = 2;
  optional bytes transaction_id = 3;
  optional FinalizeOperation finalize_operation = 4;
  repeated PostCommitHook post_commit_hooks = 5;
  optional Session session = 6;
  bool lock_mapped_range = 7;
}

One BatchCmdRequest = one raft log entry. Multiple ops in union apply atomically to the underlying Pebble batch.

2 · Headers & descriptors

message Header {
  ReplicaDescriptor replica = 1;
  uint64 range_id = 2;
  uint64 generation = 3;
  enum ConsistencyMode { LINEARIZABLE = 0; STALE = 1; RANGELEASE = 2; }
  ConsistencyMode consistency_mode = 4;
  optional Session session = 5;
  bool skip_generation_check = 6;
}

message RangeDescriptor {
  bytes start = 1; bytes end = 2;
  uint64 range_id = 3;
  repeated ReplicaDescriptor replicas = 4;
  uint64 generation = 5;
  optional uint64 last_added_replica_id = 6;
  optional int64  last_replica_added_at_usec = 7;
  repeated ReplicaDescriptor removed = 8;
  repeated ReplicaDescriptor staging = 9;
  bool deleted = 10;
}

message ReplicaDescriptor {
  uint64 range_id = 1; uint64 replica_id = 2;
  optional string nhid = 3;
}

message NodeDescriptor {
  string nhid = 1;
  string raft_address = 2;
  string grpc_address = 3;
  string zone = 4;
}

3 · Leases

message RangeLeaseRecord {
  message NodeLiveness      { int64 epoch = 1; bytes nhid = 2; }
  message ReplicaExpiration { int64 expiration = 1; bytes nhid = 2; }
  oneof value {
    NodeLiveness      node_liveness      = 1;  // normal ranges: anchored to node liveness epoch
    ReplicaExpiration replica_expiration = 2;  // meta range only: wall-clock
  }
}

4 · Transactions

message TxnRequest {
  optional bytes transaction_id = 1;
  message Statement {
    BatchCmdRequest raw_batch = 2;
    RangeDescriptor range = 3;
    repeated TransactionHook hooks = 4;
    bool range_validation_required = 5;
  }
  repeated Statement statements = 2;
}

message TxnRecord {
  TxnRequest txnRequest = 1;
  enum TxnState { UNKNOWN_STATE = 0; PENDING = 1; PREPARED = 2; COMPLETED = 3; }
  TxnState txn_state = 2;
  int64 created_at_usec = 3;
  FinalizeOperation op = 5;
}

message Session {
  bytes id = 1;
  uint64 index = 2;
  int64 created_at_usec = 5;
  optional bytes rsp_data = 4;
  optional uint64 entry_index = 6;
}

Sessions provide idempotency: the same SyncPropose retried after a leader change won't be applied twice — the replica's state machine sees the matching (id, index) and returns the cached response.

raft_service.proto internal gRPC

The internal mesh API between Store nodes. Not what clients hit.

proto/raft_service.proto — service Api
service Api {
  // Raft meta-API.
  rpc StartShard(raft.StartShardRequest)         returns (raft.StartShardResponse);
  rpc RemoveData(raft.RemoveDataRequest)         returns (raft.RemoveDataResponse);
  rpc AddReplica(raft.AddReplicaRequest)         returns (raft.AddReplicaResponse);
  rpc RemoveReplica(raft.RemoveReplicaRequest)   returns (raft.RemoveReplicaResponse);
  rpc TransferLeadership(TransferLeadershipRequest)
                                              returns (TransferLeadershipResponse);

  // Workhorse API.
  rpc SyncPropose(SyncProposeRequest) returns (SyncProposeResponse);
  rpc SyncRead(SyncReadRequest)       returns (SyncReadResponse);

  // Registry & debug.
  rpc GetRegistry(GetRegistryRequest)         returns (GetRegistryResponse);
  rpc GetRangeDebugInfo(...)                  returns (...);
  rpc AdminUpdateDescriptor(...)              returns (...);
  rpc CheckRangeOverlaps(...)                 returns (...);
}

SyncPropose (write) and SyncRead (linearizable read) are the workhorses; everything else manages cluster membership and shape.

Glossary

Alphabetical reference for every term that appears in the documentation.

TermDefinition
APIClientConnection pool of raft.service.Api gRPC clients keyed by peer address. Translates an NHID to a usable gRPC stub via the Registry. →
AutoBringupEventGossip query name used by the bringup coordinator to discover other seed nodes during initial cluster formation. Replies carry {nhid, grpcAddress}. →
BatchCmdRequestUniversal envelope for all reads, writes, admin ops and txn control directed at one range. Carries a Header, a union[] of RequestUnions, optional transaction_id, optional finalize_operation, optional post_commit_hooks, optional session. →
BringupThe initial cluster formation process. Uses gossip AutoBringupEvent queries to discover peers; the elected coordinator creates the meta range and the first partition's shards once all expected seed nodes have replied. →
CAS (compare-and-set)A RequestUnion type. Used pervasively for range-lease acquisition, node-liveness renewal, range-descriptor updates, etc. The CAS is replicated through Raft like any other write.
CAS (cache)Content Addressable Storage — Bazel's CAS, the blob storage half of the Remote Execution API. Cache type CAS in resource.ResourceName. (Contrast: AC = Action Cache.)
Consistency modeThree modes carried in the request Header: LINEARIZABLE (full ReadIndex), RANGELEASE (lease-holder serves locally), STALE (any replica, no checks). →
customTimeA per-object timestamp on GCS objects, set at write time. The bucket's DaysSinceCustomTime lifecycle rule uses it to decide deletion. Bumped on atime updates to keep hot objects alive.
DragonboatExternal Go library (github.com/lni/dragonboat/v4) providing multi-group Raft: leader election, log replication, snapshotting, membership changes, multiplexed transport. BuildBuddy builds range leases, transactions, and placement on top. →
DriverPer-store priority-queued background scheduler. Detects under/over-replication, dead replicas, oversized ranges; schedules adds/removes/splits/lease transfers. →
EventRangeUsageUpdatedIn-process event published by a Replica when its estimated disk-usage changes. Consumed by checkIfReplicasNeedSplitting (triggers split) and updateStoreUsageTag (re-gossips store usage).
FileMetadataThe value stored at each metadata key. Holds FileRecord, StorageMetadata, encryption metadata, stored_size_bytes, last_access_usec, last_modify_usec. →
FileRecordThe canonical identifier for a stored blob: {Isolation, Digest, Compressor, Encryption, DigestFunction}. The pebble key is derived from this.
FileStorerThe filestore.Store interface that wraps reading/writing blob bytes. Metacache uses only 4 methods: InlineReader, BlobReader, InlineWriter, BlobWriter.
GenerationMonotonic uint64 on every RangeDescriptor, bumped on every change. Carried in request Header; if the replica's local generation doesn't match, returns RangeNotCurrentMsg and the sender re-fetches from the meta range.
GossipHashicorp serf-based eventually-consistent state distribution. Used for: meta-range descriptor propagation, store-usage tags, NHID→address discovery, bringup coordination, per-partition usage updates.
handleMetadataMismatchMetacache function that issues a conditional DeleteRequest{MatchAtime} when a GCS read returns NotFound despite the metadata existing. Cleans up dangling pointers.
HeaderPer-request envelope inside BatchCmdRequest: {Replica, RangeId, Generation, ConsistencyMode, Session?, SkipGenerationCheck}. Used by the receiving replica for routing and fencing.
Header buildersFactory functions in raft/header/header.go for various validation strictness: MakeLinearizableWithRangeValidation, MakeLinearizableWithLeaseValidationOnly, MakeLinearizableWithoutRangeValidation.
Idempotency sessionA Session{Id, Index} attached to each BatchCmdRequest. The replica's state machine caches the response in SessionPrefix+id; replays at the same index return the cached response without re-executing.
Inline dataSmall blobs (default ≤ 1024 bytes) stored inside the FileMetadata.StorageMetadata.InlineMetadata.Data field, never sent to GCS.
IOnDiskStateMachineDragonboat interface (statemachine.IOnDiskStateMachine) implemented by Replica. Methods: Open, Update, Lookup, Sync, PrepareSnapshot, SaveSnapshot, RecoverFromSnapshot, Close.
IsolationSub-message of FileRecord namespacing a record: {cache_type, remote_instance_name, partition_id, group_id}.
LeasekeeperPer-Store. Manages one leaseAgent per range; reacts to dragonboat leader-change events and node-liveness epoch changes to acquire/drop range leases. →
LeasedRangeThe Store-side validation function called by every SyncPropose and SyncRead. Checks: range hosted here? lease held? generation matches? Returns sentinel OutOfRange errors that the sender uses to decide whether to retry the same replica, try a different one, or refresh the cache.
LINEARIZABLEConsistency mode using dragonboat ReadIndex — leader confirms it's still leader before serving. Strongest guarantee; one network round-trip.
LockMappedRangeFlag on BatchCmdRequest that, during PrepareTransaction, locks the entire range against new writes. Used during split prepare so writes to the second half can't sneak in before the descriptor change commits.
MatchAtimeOptimistic-concurrency primitive on DeleteRequest. Only deletes if the row's atime matches. Used by the eviction sampler and by handleMetadataMismatch for orphan cleanup.
Meta rangeRange with range_id = 1. Stores every other range's descriptor at \x02 || rd.end. Bootstrap-critical; never split; uses a time-based lease (not node-liveness) to break circular dependency.
MetadataServicePublic gRPC service (Get / Set / Find / Delete) exposed by every metadata server. Backed by raft. →
MetaRangeTagGossip user-event name carrying the meta range's descriptor. Re-published by any store hosting the meta range; consumed by all stores' rangecache.
NHID (NodeHost ID)Dragonboat's stable identifier for a node. Used everywhere addresses are resolved.
Node livenessOne short-lived lease record per node, stored in the meta range at SystemPrefix || nhid. Renewed every ~5s with a 9s TTL via CAS. Every range lease this node holds is invalidated when the liveness epoch advances. →
NodeRegistryDragonboat interface (raftio.INodeRegistry) mapping (rangeID, replicaID) to a peer address. BuildBuddy provides a gossip-backed implementation. →
NodeHostDragonboat's per-process handle. One per node. Owns transport, log database, runs all raft state machines.
OutOfRangeErrorSentinel error class used by the sender for retry decisions. Subtypes (string-matched on message prefix): RangeNotFoundMsg, RangeNotLeasedMsg, RangeLeaseInvalidMsg, RangeNotCurrentMsg.
PartitionLogical grouping of cache entries by group_id + remote-instance prefix. Each partition has its own range set, its own usage tracker, its own eviction LRU.
PartitionDescriptorPer-partition metadata stored in the meta range at SystemPrefix + "partition-". Carries state (INITIALIZING / INITIALIZED), first range ID, etc.
PebbleCockroachDB's LSM key-value store. One DB per Store node, multiplexed across all (range, replica) pairs via byte-prefix scoping.
PostCommitHookSide-effect attached to a BatchCmdRequest. Two types: SnapshotClusterHook (request a fresh snapshot for this shard) and StartShardHook (start a new dragonboat shard — used during split prepare).
PREPARE / COMMIT / ROLLBACKThe three phases of a 2PC transaction. FinalizeOperation on the batch selects which phase. →
RangeContiguous [start, end) byte interval in the global keyspace, served by one dragonboat shard with N replicas. Identified by range_id (uint64). Bounds change via splits (and theoretically merges, though not in current code).
Range leaseOne per range. Stored at LocalRangeLeaseKey in the range's own raft group. Acquired by the raft leader via CAS. Anchored to a node-liveness epoch (normal ranges) or a wall-clock expiration (meta range only). →
RangeCacheLocal in-process map from key-range to RangeDescriptor. Used by the sender to avoid hitting the meta range on every operation. Replaced only on higher-generation updates.
RangeDescriptorProto describing a range: {start, end, range_id, replicas[], generation, staging[], removed[], deleted}. Stored both locally on every replica and globally in the meta range at RangeMetaKey(end).
RangeMetaKey(key)Helper that prepends \x02 to a key, producing the corresponding meta-range entry key. So a range with end X is stored at meta-range key \x02X.
ReadIndexDragonboat's protocol for linearizable reads: leader commits a no-op to confirm leadership, then serves the read from local state. Used for LINEARIZABLE consistency.
RegistrySee NodeRegistry.
ReplicaOne member of a range's raft group on one node. Identified by (range_id, replica_id). Has a Go *replica.Replica implementing dragonboat's IOnDiskStateMachine. →
RequestUnionOneof inside BatchCmdRequest.union[]. Each entry is one operation (DirectWrite, CAS, Get, Set, ScanRequest, etc.). One batch can contain multiple, applied atomically.
Salt5-char random suffix appended to GCS blob names by filestore.BlobWriter. Two writes of the same digest produce two distinct GCS objects. The metadata pointer authoritatively says which is current.
SenderClient-side request router. Looks up the responsible range, tries replicas in preference order, retries on stale routing. →
SerfHashicorp's gossip implementation. Used by BuildBuddy for cluster membership + the various gossip channels.
Session (client)Per-traffic-class object (client.Session) holding an ID + monotonic index. Attached to each SyncPropose for idempotency. Stored in the state machine; periodically GC'd by deleteSessionWorker.
ShardDragonboat's term for what BuildBuddy calls a range. The dragonboat ShardID is the same uint64 as the BuildBuddy range_id.
StagingA list on RangeDescriptor: replicas being added but not yet voters. The driver moves them from Staging to Replicas after the new replica catches up to the leader's applied index. →
STALEConsistency mode: any replica with the range loaded answers from local state, no checks. Used for diagnostics and prefetching.
StorageMetadataDiscriminated union inside FileMetadata. Variants: InlineMetadata (≤ 1 KiB; bytes embedded), GCSMetadata (blob name + custom-time), PebbleMetadata + FileMetadata (used by other backends, not metacache).
StorePer-process orchestrator. Owns the dragonboat NodeHost, the local Pebble DB, every Replica, the LeaseKeeper, the Driver, the txn coordinator, the UsageTracker. →
StoreMapCluster-wide view of every node's load, capacity, aliveness. Consumed by the driver (not by routing). Updated by gossip member events + live aliveness checks.
StoreUsageTagGossip member tag carrying a base64-encoded StoreUsage proto (replica count, lease count, disk, QPS, ready bit).
SyncProposeThe "write" RPC. Goes through dragonboat consensus on the target range. Available on the public raft.service.Api service.
SyncReadThe "read" RPC. Behavior depends on consistency mode in the header: ReadIndex / lease / stale.
TxnRecordDurable record of an in-flight transaction, stored in the meta range. Drives crash recovery — any node holding the meta-range lease can replay PREPARED records' finalizes. →
UnsplittableMaxByteConstant \x04. The splitter refuses to split keys below this byte. Effectively reserves [\x01, \x04) for system / meta / local key space.
UsageTrackerPer-Store. Tracks local + global per-partition disk usage. Drives eviction (4 worker goroutines per partition). Cross-cluster aggregation via gossip NodePartitionUsageEvent. →
